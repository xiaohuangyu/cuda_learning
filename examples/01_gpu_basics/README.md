# Chapter 01: GPU基础 - CPU vs GPU性能对比

## 概述

本示例演示了相同的计算任务在CPU和GPU上的执行时间对比，帮助理解GPU并行计算的基本概念和优势。

## 示例文件

- `cpu_vs_gpu.cu` - CPU与GPU向量加法对比程序

## 核心概念

### CPU vs GPU 架构差异

| 特性 | CPU | GPU |
|------|-----|-----|
| 核心数量 | 少（通常4-64个） | 多（数千个） |
| 单核性能 | 强 | 较弱 |
| 并行能力 | 有限 | 极强 |
| 适用场景 | 复杂逻辑、串行任务 | 大规模数据并行任务 |

### CUDA编程模型要点

1. **主机（Host）**：CPU及其内存
2. **设备（Device）**：GPU及其内存
3. **核函数（Kernel）**：在GPU上并行执行的函数
4. **线程层次结构**：Grid → Block → Thread

## 编译与运行

```bash
# 创建构建目录
mkdir build && cd build

# 配置项目
cmake ..

# 编译
make

# 运行
./cpu_vs_gpu
```

## 预期输出

程序将显示：
1. 内存分配信息
2. CPU执行时间
3. GPU执行时间（包括数据传输）
4. 结果验证
5. 性能加速比

## 学习要点

1. 理解CPU串行执行与GPU并行执行的区别
2. 掌握CUDA内存管理：`cudaMalloc`、`cudaMemcpy`
3. 了解核函数调用语法：`kernel<<<grid, block>>>(args)`
4. 理解线程索引计算方式

## 注意事项

- 首次运行时，数据传输开销可能较大
- 数据量越大，GPU优势越明显
- 简单运算可能无法充分展示GPU优势
